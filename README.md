#arcenesis-magnit-uplift
hurdle t-learner на catboost.

## запуск
```bash
uv sync
uv run python -m src.train --train data/train.parquet --out artifacts/
uv run python -m src.predict --test data/test.parquet --models artifacts/ --out predictions.csv
```

## внутри
4 catboost модели:
- p_treat, p_ctrl: купит/не купит
- e_treat, e_ctrl: сумма при условии покупки
uplift = p_treat * e_treat - p_ctrl * e_ctrl

## валидация
5-fold stratified kfold по treatment_flg * 2 + is_buyer, метрика uplift@10 с бутстрэпом(нижняя граница 95% ди).

## Методология и архитектура решения

### 1. Архитектура модели (Hurdle Uplift Model)

Распределение целевой переменной (выручки) имеет сильный спад в районе нуля (большинство пользователей ничего не покупают). Использование классических линейных моделей или стандартной регрессии в лоб здесь неэффективно. 

Решение построено на **Hurdle-архитектуре (двухстадийной модели)**, которая обучается независимо для тестовой (Treatment) и контрольной (Control) групп. Пайплайн состоит из 4 моделей CatBoost:

1.  **$P_{treat}(X)$** (CatBoostClassifier) — вероятность совершения покупки в группе Treatment: 
    $$P(Spend > 0 \mid X, Treatment)$$
2.  **$P_{ctrl}(X)$** (CatBoostClassifier) — вероятность совершения покупки в группе Control:
    $$P(Spend > 0 \mid X, Control)$$
3.  **$E_{treat}(X)$** (CatBoostRegressor) — ожидаемый логарифм выручки в Treatment при условии совершения покупки (обучается только на пользователях с выручкой > 0):
    $$E(\ln(Spend + 1) \mid X, Treatment, Spend > 0)$$
4.  **$E_{ctrl}(X)$** (CatBoostRegressor) — ожидаемый логарифм выручки в Control при условии совершения покупки (обучается только на пользователях с выручкой > 0):
    $$E(\ln(Spend + 1) \mid X, Control, Spend > 0)$$

**Расчет итогового аплифта (Inference):**
Для каждого пользователя скор рассчитывается как разница математических ожиданий выручки с обратной трансформацией логарифма (`expm1`):

$$Uplift(X) = P_{treat}(X) \cdot (\exp(E_{treat}(X)) - 1) - P_{ctrl}(X) \cdot (\exp(E_{ctrl}(X)) - 1)$$

---

### 2. Схема валидации (Validation Strategy)

Для локального тестирования моделей и защиты от переобучения выстроена следующая схема:

*   **Кросс-валидация:** 5-Fold CV.
*   **Схема стратификации:** Разделение на фолды происходит сбалансировано по целевому признаку группы и факту покупки:
    $$Stratification\_Target = Treatment\_Flg \cdot 2 + (Spend > 0)$$
    Это гарантирует одинаковую долю целевой и контрольной групп, а также платящих пользователей в каждом фолде.
*   **Локальная метрика (Uplift@10 LB):**
    В качестве ключевой метрики используется **нижняя граница 95% доверительного интервала** для метрики Uplift@10, полученная методом бутстрапа (количество подвыборок $N = 1000$). Использование именно нижней границы (Lower Bound) делает оценку консервативной и защищает от случайных выбросов при валидации.

### 3. Технологический стек

*   **Язык программирования:** Python 3.11
*   **Управление средой и зависимостями:** `uv` (гарантирует детерминизм и скорость сборки окружения в 10-100 раз быстрее стандартного `pip`)
*   **Обработка данных:** `Polars` (эффективное использование многопоточности и метаданных без перегрузки RAM)
*   **Моделирование:** `CatBoost`
*   **Инфраструктура:** Docker (multi-stage сборка для оптимизации размера образа), Taskfile (опциональная автоматизация запуска)

## Воспроизведение решения

В этом разделе описаны системные требования и инструкции по сборке, валидации и ручному запуску пайплайна

### 1. Системные требования

Для воспроизведения должны быть установлены:
*   **Docker Daemon** (версии 20.10.0 или новее)
*   **Python 3.11** (необходим для локального запуска скриптов валидации)
*   Менеджер пакетов **`uv`** (рекомендуется для быстрой установки зависимостей, но не обязательно)


### 2. Быстрый автоматический запуск

В репозитории подготовлен автоматический скрипт тестирования `scripts/test_pipeline.py`. Он в один клик собирает образ, генерирует предсказания в контейнере, тестирует их на соответствие схеме данных, отсутствие пропусков и проверяет алгоритм на детерминизм (повторяемость результатов)

Для запуска автоматического теста выполните команду из корня проекта:

**С использованием `uv`:**
```bash
uv run python scripts/test_pipeline.py
```

**С использованием стандартного интерпретатора Python:**
```bash
python scripts/test_pipeline.py
```



### 3. Опционально: Использование Taskfile (Для продвинутых пользователей)

Если в вашей системе установлен инструмент автоматизации **Go-Task**, вы можете запустить весь цикл тестов (сборка + запуск + валидация) одной короткой командой:

```bash
task test
```
*Все инструкции и зависимости шагов уже описаны в файле `Taskfile.yaml`.*


### 4. Ручной запуск (Сборка и инференс)

Если вам необходимо выполнить сборку образа и запуск инференса вручную шаг за шагом, следуйте инструкциям ниже.

#### Шаг 4.1. Сборка Docker-образа
Соберите легковесный Docker-образ на базе multi-stage сборки с тегом `magnit-uplift`. Выполните команду из корня проекта:

```bash
docker build -t magnit-uplift .
```

#### Шаг 4.2. Запуск контейнера для получения предсказаний

Для работы инференса необходимо примонтировать локальные папки с данными и для сохранения результатов. 

Выполните команду запуска в соответствии с вашей операционной системой. По умолчанию скрипт будет искать файл `/app/data/test.parquet` внутри контейнера и сохранит результат в `/app/artifacts/predictions.csv`.

**Для Linux / macOS (Bash/Zsh):**
```bash
docker run --rm \
  -v "$(pwd)/data:/app/data" \
  -v "$(pwd)/artifacts:/app/artifacts" \
  magnit-uplift \
  --test /app/data/test.parquet \
  --out /app/artifacts/predictions.csv
```

**Для Windows (PowerShell):**
```powershell
docker run --rm `
  -v "${PWD}/data:/app/data" `
  -v "${PWD}/artifacts:/app/artifacts" `
  magnit-uplift `
  --test /app/data/test.parquet `
  --out /app/artifacts/predictions.csv
```

*Примечание:* Если вы хотите запустить контейнер в режиме заглушки со случайными числами (для проверки работоспособности без обученных весов моделей), добавьте флаг `--stub` в конец любой из команд выше.
